# 실습 개요

## 실습 시나리오

> **"AI 회의록 요약기를 만들자"**

이 실습에서는 슬래시 커맨드를 사용해 <strong>기획부터 디자인, 구현, 검증, 커밋까지의 전체 워크플로우</strong>를 체험합니다.

회의 텍스트를 입력하면 AI가 자동으로 요약하고, 액션 아이템을 추출하는 실용적인 마이크로 앱을 만들어봅니다.

## 워크플로우 흐름

```
 ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
 │  /brainstorm │──→│    /plan     │──→│   /design   │──→│ /orchestrate│──→│   /review   │──→│   /commit   │
 │  아이디어 발산│   │  작업 계획   │   │  UI 시안    │   │  자동 실행   │   │  품질 검증   │   │ 깔끔한 커밋  │
 └─────────────┘   └─────────────┘   └─────────────┘   └─────────────┘   └─────────────┘   └─────────────┘
       ↓                  ↓                 ↓                  ↓                 ↓                 ↓
  디자인 문서          plan.json       Stitch 시안 링크     완성된 코드       검증 보고서       커밋 히스토리
  (design.md)        (태스크 분해)     (사용자 컨펌)        (시안 기반 구현)   (보안/성능/접근)   (기능별 분리)
```

## 각 단계의 역할

| 단계 | 슬래시 커맨드 | 에이전트 | 산출물 |
|------|-------------|----------|--------|
| 1단계 | `/brainstorm` | oma-brainstorm | 설계 문서 (`docs/plans/*.md`) |
| 2단계 | `/plan` | oma-pm | 실행 계획 (`plan.json`) |
| 3단계 | `/design` | oma-design | Stitch UI 시안 + 디자인 토큰 |
| 4단계 | `/orchestrate` | oma-orchestrator | 구현된 코드 |
| 5단계 | `/review` | oma-qa | 검증 보고서 (보안·성능·접근성) |
| 6단계 | `/commit` | oma-commit | Conventional Commits |

## 추천 워크플로우 순서

::: details 📋 상황별 추천 패턴

프로젝트 규모와 목적에 따라 다른 조합을 쓸 수 있습니다:

| 상황 | 워크플로우 순서 | 특징 |
|------|---------------|------|
| **간단한 작업** | `/plan` → `/coordinate` → `/commit` | 가이드형, 빠른 실행 |
| **품질 중시** | `/brainstorm` → `/plan` → `/ultrawork` → `/commit` | 17단계 중 11개 리뷰, 가장 엄격 |
| **디자인 중시** | `/brainstorm` → `/plan` → `/design` → `/orchestrate` → `/commit` | Stitch MCP로 시안 선 확정 |
| **대규모 병렬** | `/plan` → `/orchestrate` → `/review` → `/commit` | CLI 자동 스폰, 실시간 모니터링 |

> **이 실습에서는 "디자인 중시" 패턴**에 `review`를 추가한 풀코스를 체험합니다.

:::

## 핵심 포인트

::: tip 🎯 왜 이 순서인가?

**"코드부터 쓰지 마라."**

1. `/brainstorm` → "무엇을 만들 것인가?" 를 명확히
2. `/plan` → "어떻게 나눠서 만들 것인가?" 를 구조화
3. `/design` → "어떻게 보일 것인가?" 를 시각적으로 확정
4. `/orchestrate` → "AI 팀이 실행" 자동화
5. `/review` → "품질이 괜찮은가?" 검증
6. `/commit` → "깔끔하게 기록" 마무리

이 6단계를 건너뛰고 바로 코딩하면, AI도 사람도 길을 잃습니다.
:::
