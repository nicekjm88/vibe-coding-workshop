# /review — 품질 검증

## 목표

QA 에이전트(`oma-qa`)가 구현된 코드를 <strong>4가지 축으로 검증</strong>합니다: 보안, 성능, 접근성, 코드 품질.

## 실행 방법

```
/review
orchestrate 결과물을 리뷰해줘
```

## 진행 과정

### Step 1: 보안 감사 (OWASP Top 10)

QA 에이전트가 <strong>보안 취약점</strong>을 자동으로 스캔합니다:

```
🤖 QA 에이전트: "보안 감사를 시작합니다...

 🔒 OWASP Top 10 체크리스트
 ├── ✅ A01: 접근 제어 — API 라우트 인증 미들웨어 적용됨
 ├── ✅ A02: 암호화 — HTTPS 강제, 세션 토큰 관리 정상
 ├── ⚠️ A03: 인젝션 — raw_text 입력에 sanitize 누락
 ├── ✅ A07: 인증 — 세션 기반 인증 정상
 └── ✅ A09: 로깅 — 에러 핸들링 구현됨

 ⚠️ 발견 1건: XSS 방지를 위한 입력 sanitize 필요
    → src/lib/sanitize.ts 자동 생성 중..."
```

### Step 2: 성능 분석

```
🤖 QA 에이전트: "성능 분석 결과:

 📊 번들 사이즈
 ├── Total: 142KB (gzip)
 ├── React: 42KB
 ├── SQLite3 (better-sqlite3): 0KB (서버사이드)
 └── App Code: 42KB

 ⚡ 최적화 제안:
 └── Gemini API 호출 디바운싱 적용 → 불필요한 요청 방지

 자동 수정을 적용할까요?"

👤 사용자: "적용해줘"
```

### Step 3: 접근성 검증 (WCAG 2.1 AA)

```
🤖 QA 에이전트: "접근성 검증 결과:

 ♿ WCAG 2.1 AA 체크리스트
 ├── ✅ 키보드 네비게이션 — Tab 순서 정상
 ├── ✅ 색 대비 — AAA 기준 충족 (7:1)
 ├── ⚠️ ARIA 라벨 — 요약 버튼에 aria-label 누락
 └── ✅ 화면 리더 — 시맨틱 HTML 사용

 ⚠️ 발견 1건: 자동 수정 적용됨
    → <button aria-label='회의 내용 요약하기'>요약</button>"
```

### Step 4: 코드 품질 리뷰

```
🤖 QA 에이전트: "코드 품질 종합 보고:

 📋 리뷰 결과
 ┌────────────┬────────┬─────────┐
 │ 항목        │ 상태   │ 점수     │
 ├────────────┼────────┼─────────┤
 │ 보안        │ ✅ 통과 │ 95/100  │
 │ 성능        │ ✅ 통과 │ 92/100  │
 │ 접근성      │ ✅ 통과 │ 98/100  │
 │ 코드 품질   │ ✅ 통과 │ 90/100  │
 └────────────┴────────┴─────────┘
 
 종합 점수: 94/100 (Grade A)
 
 자동 수정: 2건 적용됨
 수동 확인 필요: 0건"
```

## 산출물

이 단계가 끝나면 <strong>검증 완료된 코드</strong>가 됩니다:

```
자동 수정된 파일
├── src/lib/sanitize.ts        (신규: XSS 방지)
├── src/lib/db.ts              (수정: 쿼리 최적화)
└── src/components/SummaryButton.tsx (수정: ARIA 라벨)
```

::: info 🎯 핵심 메시지
**"코드가 돌아간다 ≠ 좋은 코드다."**

`/review`는 사람이 놓치기 쉬운 보안·성능·접근성을 자동으로 잡아줍니다.
코드 리뷰를 AI에게 맡기면, 일관된 기준으로 매번 검증할 수 있습니다.
:::
