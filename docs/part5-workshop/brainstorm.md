# /brainstorm — 아이디어 발산

## 목표

브레인스토밍 에이전트(`oma-brainstorm`)와 대화하며 <strong>아이디어를 구체화</strong>하고, 설계 문서를 만듭니다.

## 실행 방법

AI IDE에서 다음과 같이 입력합니다:

```
/brainstorm
회의록을 입력하면 AI가 자동으로 요약하고 액션 아이템을 추출해주는 앱을 만들고 싶어.
Next.js + FastAPI + SQLite3 + Gemini API로 구성하려 해.
```

## 진행 과정

### Step 1: 프로젝트 컨텍스트 탐색

에이전트가 먼저 <strong>현재 프로젝트의 상태</strong>를 파악합니다:
- 기존 코드가 있는지
- 어떤 프레임워크를 쓰고 있는지
- 관련된 패턴이나 규칙이 있는지

```
🤖 에이전트: "프로젝트를 분석했습니다. 현재 빈 프로젝트이며,
             React + SQLite3 + Gemini API 구성을 원하시는군요."
```

### Step 2: 명확화 질문

에이전트가 <strong>하나씩</strong> 질문을 합니다:

```
🤖 에이전트: "회의록 입력 방식은 어떤 걸 선호하시나요?
             A) 텍스트 직접 입력 (복사-붙여넣기)
             B) 음성 녹음 후 STT 변환
             C) 파일 업로드 (txt, docx)"

👤 사용자: "A"

🤖 에이전트: "요약 결과에 포함할 항목은?
             A) 요약만
             B) 요약 + 액션 아이템
             C) 요약 + 액션 아이템 + 담당자 배정"

👤 사용자: "C"
```

::: tip 💡 포인트
에이전트가 질문을 <strong>한 번에 하나씩</strong> 하는 것에 주목하세요.
한꺼번에 10개를 물어보면 사용자가 압도당합니다.
:::

### Step 3: 접근 방식 제안

충분한 정보가 모이면, <strong>2~3가지 접근 방식</strong>을 제안합니다:

```
🤖 에이전트: "3가지 접근 방식을 제안드립니다:

 🅰 심플 SPA형 (권장)
    - React SPA + SQLite3 + Gemini API 직접 호출
    - 장점: 설치 간단, 로컬에서 바로 실행
    - 단점: 멀티유저 미지원
    - 난이도: S

 🅱 Express 서버형
    - React + Express + SQLite3 (서버사이드 AI 호출)
    - 장점: API 키 보안, 서버 로직 분리
    - 단점: 서버 설정 필요
    - 난이도: M

 🅒 풀스택 분리형
    - React + FastAPI + SQLite3 + Gemini
    - 장점: 백엔드 완전 제어, 프롬프트 관리
    - 단점: 학습 범위 넓음
    - 난이도: L"

👤 사용자: "🅰로 가자"
```

### Step 4: 상세 설계

선택된 접근 방식을 <strong>섹션별로</strong> 상세 설계합니다:

```
🤖 에이전트: "아키텍처 설계를 보여드립니다:

 ┌── App ──────────────────────────┐
 │  ┌── MeetingInput ────────────┐ │
 │  │  텍스트 입력 영역           │ │
 │  │  [요약하기] 버튼            │ │
 │  └────────────────────────────┘ │
 │  ┌── SummaryView ─────────────┐ │
 │  │  AI 요약 결과 표시          │ │
 │  └────────────────────────────┘ │
 │  ┌── ActionItems ─────────────┐ │
 │  │  액션 아이템 + 담당자 목록   │ │
 │  └────────────────────────────┘ │
 │  ┌── MeetingHistory ──────────┐ │
 │  │  과거 회의록 목록            │ │
 │  └────────────────────────────┘ │
 └──────── SQLite3 + Gemini ───────┘

 승인하시겠습니까?"

👤 사용자: "좋아, 진행해"
```

### Step 5: 설계 문서 저장

승인된 설계를 <strong>파일로 저장</strong>합니다:

```
🤖 에이전트: "설계 문서를 저장했습니다:
             📄 docs/plans/meeting-summarizer-design.md
             
             다음 단계로 /plan을 실행하세요."
```

## 산출물

이 단계가 끝나면 <strong>설계 문서</strong>가 생성됩니다:

```
docs/plans/meeting-summarizer-design.md
├── 프로젝트 개요
├── 선택된 접근 방식
├── 아키텍처 설계
├── 컴포넌트 구조
├── 데이터 모델
└── 에러 핸들링 전략
```

::: info 🎯 핵심 메시지
**"AI가 바로 코드를 쓰는 게 아니라, 먼저 사고하고 합의한다."**

이 과정이 없으면 AI는 자기 마음대로 만들고, 사용자는 나중에 "이게 아닌데…"를 반복합니다.
:::
