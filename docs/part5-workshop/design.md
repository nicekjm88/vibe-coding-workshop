# /design — UI 시안 생성

## 목표

디자인 에이전트(`oma-design`)가 **Stitch MCP**를 활용하여 <strong>코드 작성 전에 UI 시안을 생성</strong>하고, 사용자의 승인을 받습니다.

## 왜 이 단계가 필요한가?

::: warning ⚠️ 디자인 없이 코딩하면 생기는 일
- 프론트엔드 에이전트가 **자의적으로** 화면을 구성
- 완성 후 "이건 내가 원한 게 아닌데…" 반복
- 수정 비용이 구현 후에 급격히 증가
:::

`/design` 단계를 추가하면:
- **구현 전**에 시각적으로 화면을 확인
- 수정은 **텍스트 프롬프트 한 줄**로 가능
- 확정된 디자인이 프론트엔드 에이전트의 **구현 가이드**가 됨

## 실행 방법

```
/design
plan에서 나온 회의록 요약기의 UI 시안을 만들어줘
```

## 진행 과정

### Step 1: Stitch 프로젝트 생성

에이전트가 <strong>Stitch MCP</strong>를 호출하여 디자인 작업 환경을 만듭니다:

```
🤖 디자인 에이전트: "Stitch 프로젝트를 생성합니다...
                    ✅ 프로젝트 'Meeting Summarizer' 생성 완료"
```

### Step 2: 디자인 시스템 설정

프로젝트에 <strong>디자인 토큰</strong>(색상, 폰트, 모양)을 적용합니다:

```
🤖 디자인 에이전트: "디자인 시스템을 설정합니다:
                    - 컬러: Modern Dark 프리셋
                    - 폰트: Inter
                    - 라운딩: Rounded
                    ✅ 디자인 시스템 적용 완료"
```

### Step 3: 화면 시안 생성

설계 문서를 기반으로 <strong>각 화면을 텍스트 프롬프트로 생성</strong>합니다:

```
🤖 디자인 에이전트: "2개 화면을 생성합니다:

                    📱 화면 1: 메인 (입력 + 히스토리)
                    '왼쪽 사이드바에 과거 회의록 목록,
                     오른쪽에 텍스트 입력창과 요약 버튼'

                    📱 화면 2: 결과 (요약 + 액션 아이템)
                    '핵심 요약 카드, 액션 아이템 체크리스트,
                     담당자 배지가 표시된 결과 뷰'

                    ✅ 시안 생성 완료"
```

### Step 4: 사용자 컨펌

Stitch 프로젝트 <strong>링크를 공유</strong>합니다:

```
🤖 디자인 에이전트: "시안을 확인해주세요:
                    🔗 https://stitch.google.com/projects/xxxxx

                    수정이 필요하시면 말씀해주세요.
                    예: '버튼 색상을 파란색으로 변경해줘'
                    예: '사이드바를 좀 더 좁게 해줘'"

👤 사용자: "액션 아이템에 우선순위 뱃지도 추가해줘"

🤖 디자인 에이전트: "수정 반영 완료. 다시 확인해주세요:
                    🔗 https://stitch.google.com/projects/xxxxx"

👤 사용자: "좋아, 확정!"
```

### Step 5: 디자인 확정 및 저장

승인된 디자인 정보를 <strong>파일로 저장</strong>합니다:

```
🤖 디자인 에이전트: "디자인을 확정했습니다:
                    📄 docs/plans/meeting-summarizer-screens.md
                    
                    다음 단계로 /orchestrate를 실행하세요.
                    프론트엔드 에이전트가 이 시안을 기반으로 구현합니다."
```

## 산출물

이 단계가 끝나면 <strong>확정된 디자인 가이드</strong>가 생성됩니다:

```
docs/plans/meeting-summarizer-screens.md
├── Stitch 프로젝트 링크
├── 디자인 토큰 (색상, 폰트, 라운딩)
├── 화면별 구성 설명
└── 사용자 수정 이력
```

::: info 🎯 핵심 메시지
**"코드보다 시안이 먼저. 수정은 프롬프트 한 줄이면 된다."**

Stitch MCP 덕분에:
- 디자인 도구를 따로 열 필요 없이 AI가 시안을 생성
- 텍스트로 수정 요청 → 즉시 반영
- 확정된 시안이 구현의 정확한 청사진이 됨
:::
