## 코딩이란 무엇인가?

<strong>코딩</strong>은 사람이 컴퓨터에게 명령을 내리는 행위입니다.

컴퓨터는 사람의 말을 이해하지 못하기 때문에, 사람이 컴퓨터가 이해하는 <strong>프로그래밍 언어</strong>(Python, JavaScript 등)로 명령을 작성해야 했습니다.

```python
# 예시: "사용자 목록을 가져와서 화면에 보여줘"를 코드로 표현하면
users = database.get_all_users()
for user in users:
    print(user.name)
```

## 바이브 코딩은 뭐가 다른가?

바이브 코딩에서는 이 과정이 극적으로 바뀝니다:

```
사람: "사용자 목록을 가져와서 화면에 예쁘게 보여줘"
  ↓
AI: 코드를 자동 생성
  ↓
결과: 동작하는 웹 페이지
```

## 건축에 비유하면

| 역할 | 전통 개발 | 바이브 코딩 |
|------|----------|-----------|
| 설계 | 건축가 (사람) | 건축주 (사람) |
| 시공 | 시공사 (사람) | **AI** |
| 감리 | 감리사 (사람) | 사람 + AI |

과거에는 <strong>건축가(개발자)</strong>가 도면을 그리고 <strong>시공사(개발자)</strong>가 직접 지어야 했습니다.

이제는 <strong>건축주(누구나)</strong>가 "이런 집 짓고 싶어"라고 말하면, <strong>AI가 도면도 그리고 시공도 합니다.</strong>

하지만 중요한 점이 있습니다:

::: warning ⚠️ 핵심 포인트
"아무렇게나 말하면 아무렇게나 지어진다."

건축주라도 <strong>자신이 원하는 것을 명확히 전달하는 능력</strong>이 필요합니다.
이것이 바로 <strong>바이브 코딩 엔지니어링</strong>이 필요한 이유입니다.
:::

## 비개발자가 알아야 할 것

바이브 코딩 시대에 비개발자에게 필요한 역량:

1. **문제 정의 능력** — 무엇을 만들고 싶은지 명확히 설명
2. **결과 검증 능력** — AI가 만든 결과가 맞는지 판단
3. **도구 활용 능력** — AI 코딩 도구 기본 사용법
4. **협업 능력** — 개발자와 AI 사이의 가교 역할
