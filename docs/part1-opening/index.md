# 바이브 코딩이란?

> "코딩의 분위기에 몸을 맡기고, 지수적으로 발전하는 LLM에게 코드를 맡기면 된다. 코드가 실제로 작동하는지만 보면 되고, 코드를 읽을 일은 거의 없다."
>
> — **Andrej Karpathy**, 2025년 2월

## 바이브 코딩의 정의

<strong>바이브 코딩(Vibe Coding)</strong>이란, 사람이 자연어로 원하는 기능을 설명하면 AI가 코드를 생성하는 새로운 개발 패러다임입니다.

기존 개발 방식과의 차이를 비교해보면:

| 구분 | 전통적 코딩 | 바이브 코딩 |
|------|-----------|-----------|
| 입력 | 프로그래밍 언어 | 자연어(한국어, 영어 등) |
| 주체 | 사람이 직접 코드 작성 | AI가 코드 생성, 사람이 검토 |
| 필요 역량 | 문법, 알고리즘, 프레임워크 지식 | 문제 정의, 요구사항 전달 능력 |
| 속도 | 시간~일 단위 | 분~시간 단위 |

## 왜 "바이브"인가?

Karpathy가 사용한 "vibe"라는 단어는 의도적입니다:

- 코드의 세부 문법에 집착하지 않고
- AI에게 <strong>분위기(vibe)</strong>를 전달하면
- AI가 알아서 구현해준다

즉, 개발의 초점이 <strong>"어떻게(How) 코드를 짤 것인가"</strong>에서 <strong>"무엇을(What) 만들 것인가"</strong>로 이동한 것입니다.

## 이 워크숍에서 배울 것

이 워크숍에서는 바이브 코딩이 단순한 유행어가 아니라, <strong>3세대에 걸쳐 진화한 엔지니어링 분야</strong>임을 이해하게 됩니다:

1. **1세대 — 프롬프트 엔지니어링**: AI에게 잘 물어보는 기술
2. **2세대 — 컨텍스트 엔지니어링**: AI에게 맥락을 심어주는 기술
3. **3세대 — 하네스 엔지니어링**: AI 팀을 구성하고 협업시키는 기술

그리고 3세대 하네스 엔지니어링의 대표 도구인 <strong>oh-my-agent</strong>를 직접 실습합니다.
