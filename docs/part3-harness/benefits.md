| 이점 | 설명 | 수치적 효과 |
|------|------|------------|
| 🎯 **역할 전문화** | 각 에이전트가 자기 도메인에만 집중 | 결과 품질 향상 |

## 상세 설명

### 🎯 역할 전문화

2세대에서는 하나의 AI에게 모든 것을 맡겼습니다:

```
2세대: "프론트엔드도 해줘, 백엔드도 해줘, DB도 해줘, 리뷰도 해줘"
→ 모든 걸 어중간하게 수행
```

3세대에서는 각 에이전트가 <strong>자기 전문 분야에만 집중</strong>합니다:

```
3세대: oma-frontend → React, TypeScript, CSS만 전담
       oma-backend → API, 인증, 서버만 전담
       oma-db     → 스키마, 쿼리, 마이그레이션만 전담
```

### 💰 토큰 효율의 비밀

oh-my-agent는 <strong>2레이어 스킬 설계</strong>로 토큰을 절약합니다:

```
Layer 1: SKILL.md (가벼운 라우팅 정보)
         → 항상 로드됨 (수백 토큰)
         
Layer 2: 상세 스킬 파일들
         → 필요할 때만 로드됨 (수천 토큰)
```

만능 AI 방식은 매번 전체 맥락을 보내야 하지만, 하네스 방식은 <strong>해당 에이전트에게 필요한 맥락만</strong> 전송합니다.

### 🔄 재현성

워크플로우가 <strong>코드로 정의</strong>되어 있으므로:

- 같은 워크플로우는 항상 같은 프로세스를 따름
- 팀원이 바뀌어도 동일한 품질 보장
- "저번에 잘 됐는데 이번엔 왜 안 돼?" 문제 해결

### 📦 이식성

```bash
# 다른 프로젝트에 하네스 적용
cp -r .agents/ ../new-project/

# 또는 oh-my-agent로 한 줄 설치
bunx oh-my-agent
```

IDE, AI 모델, 프로젝트에 종속되지 않는 <strong>포터블 하네스</strong>입니다.
