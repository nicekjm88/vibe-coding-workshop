# 3세대 이야기

## "AI 하나로 다 하면 왜 망하는가?"

다음 상황을 상상해보세요:

```
당신: "유저 인증이 있는 TODO 앱을 만들어줘.
      React + Supabase로. 예쁘게."

AI: (3000줄의 코드를 한꺼번에 생성)
    → 프론트엔드, 백엔드, DB가 뒤섞여 있음
    → 인증 로직에 보안 허점
    → CSS가 깨져 있음
    → 테스트 코드 없음

당신: "...고쳐줘"

AI: (또 3000줄을 생성, 이전 맥락 절반을 까먹음)
```

**이것이 바이브 코딩의 현실입니다.** AI 하나에 모든 것을 맡기면, 아래와 같은 문제가 발생합니다:

| 문제 | 원인 |
|------|------|
| 🧠 맥락 손실 | 대화가 길어지면 초반 내용을 잊음 |
| 🎭 역할 혼동 | 프론트, 백엔드, DB를 구분하지 못함 |
| 💸 토큰 낭비 | 불필요한 맥락까지 매번 전달 |
| 🔄 일관성 부족 | 같은 요청에도 다른 결과 |
| 🛡️ 품질 무보증 | 보안, 성능, 접근성 검증 없음 |

이 문제들을 해결하기 위해, 바이브 코딩은 <strong>3세대에 걸쳐 진화</strong>했습니다.

## 진화의 흐름

```
1세대                2세대                3세대
프롬프트              컨텍스트              하네스
엔지니어링             엔지니어링             엔지니어링
  │                   │                   │
  │  "잘 물어보자"      │  "맥락을 줘 보자"     │  "팀을 꾸리자"
  │                   │                   │
  ▼                   ▼                   ▼
AI에게               AI에게               AI 여러 명에게
최적의 질문           프로젝트 맥락          역할을 나눠
던지기               심어주기              협업시키기
```

다음 페이지에서 각 세대를 자세히 알아보겠습니다.
