# 3세대: 하네스 엔지니어링

> **스토리**: "AI에게 팀을 만들어주자"

## 정의

<strong>하네스 엔지니어링(Harness Engineering)</strong>이란, 여러 전문 AI 에이전트를 하나의 <strong>체계(harness)</strong>로 묶어 <strong>역할 기반으로 협업</strong>시키는 기술입니다.

"Harness"는 원래 "마구(말에 채우는 장비)"를 뜻합니다. 여러 마리의 말을 하나의 마차에 연결해 <strong>함께 달리게 하는 장치</strong> — 그것이 하네스입니다.

## 핵심 전환

```
2세대: 한 명의 뛰어난 AI         →    3세대: 전문가 AI 팀
                                  
┌─────────────┐              ┌──────┐  ┌──────┐  ┌──────┐
│   만능 AI    │              │  PM  │  │  FE  │  │  BE  │
│  FE+BE+DB   │      →       │Agent │  │Agent │  │Agent │
│  +QA+Design │              └──────┘  └──────┘  └──────┘
└─────────────┘              ┌──────┐  ┌──────┐  ┌──────┐
                             │  DB  │  │  QA  │  │Debug │
                             │Agent │  │Agent │  │Agent │
                             └──────┘  └──────┘  └──────┘
```

## 핵심 개념

### 1. 역할 분리 (Role Separation)

각 AI 에이전트가 <strong>하나의 전문 역할</strong>만 수행합니다:

| 에이전트 | 역할 | 전문 지식 |
|----------|------|----------|
| PM | 요구사항 분석, 태스크 분해 | 프로젝트 관리, 우선순위 |
| Frontend | UI 구현 | React, CSS, 접근성 |
| Backend | API 개발 | 서버, 인증, 비즈니스 로직 |
| DB | 스키마 설계 | SQL, 인덱싱, 마이그레이션 |
| QA | 품질 검증 | 보안, 성능, 테스트 |
| Debug | 버그 수정 | 근본 원인 분석, 회귀 테스트 |

### 2. 워크플로우 오케스트레이션

에이전트들이 <strong>정해진 순서와 규칙</strong>에 따라 협업합니다:

```
/brainstorm  →  /plan  →  /orchestrate  →  /review
   (기획)        (계획)      (실행)          (검증)
```

### 3. 스킬 시스템 (Skills)

각 에이전트가 가진 <strong>재사용 가능한 전문 지식</strong>입니다:

```
.agents/skills/
├── oma-frontend/SKILL.md    # React, Next.js, Tailwind 전문
├── oma-backend/SKILL.md     # API, 인증, 서버 전문
├── oma-db/SKILL.md          # 스키마, 쿼리, 인덱싱 전문
└── oma-qa/SKILL.md          # 보안, 성능, 접근성 전문
```

### 4. 멀티 벤더 지원

하나의 하네스로 <strong>여러 AI 모델</strong>을 혼용할 수 있습니다:
- Frontend → Claude (UI 구현에 강함)
- Backend → Gemini (코드 분석에 강함)
- QA → GPT (리뷰에 강함)

## 3세대 비교 정리

| | 1세대 프롬프트 | 2세대 컨텍스트 | 3세대 하네스 |
|--|:---:|:---:|:---:|
| 핵심 질문 | "뭐라고 물어볼까?" | "뭘 알려줄까?" | "팀을 어떻게 꾸릴까?" |
| 작업 단위 | 단일 프롬프트 | 프로젝트 전체 | 전문 에이전트 팀 |
| 비유 | 프리랜서에게 잘 설명 | 프리랜서에게 매뉴얼 제공 | 전문 팀 구성 |
| 확장성 | ❌ | △ | ⭐ |
| 재현성 | ❌ | △ | ⭐ |
| 토큰 효율 | ❌ | ❌ | ⭐ |
| 품질 보증 | ❌ | △ | ⭐ |
